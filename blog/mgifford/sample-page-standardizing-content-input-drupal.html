Sample Page - Standardizing Content Input With Drupal

By:

mike

on

February 15, 2008

Since we started developing Content Management Systems (CMS) in 2000 we have wanted to give our clients control of their content. Since that time we have found that most organizations do not have very fancy formatting requirements and just need a few key systems that people can use which allow them to mark up their documents in a way that makes it easier for everyone to read.

The standards have changed in the last eight years, but the best practices today use style sheets and semantic html to attach meaning to the markup. This allows both search engines and other screen readers to evaluate what items are more or less important. Headers are the most important titles in your page, with H1 being the most important and H6 being the least. Using headers to increase the size of the fonts that you want to use is therefore a problem as it blurs the meaning.

We generally use the WYSIWYG (What You See Is What You Get) editor TinyMCE to provide an easy way for our clients to markup their text. With this we can limit wich tools are available and so strongly encourage our clients to use semantic markup when designing their sites. By default we limit the number of options that our clients have so that it provides them the best results.

 

 

It is useful to have a style guide like this that allows us to see the range of formatting options that we recommend for our clients..
